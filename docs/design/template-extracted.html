Aperture
How it works Features
self-hosted · single Go binary

Your agents talk to the cloud.
Know what they say.

Aperture is a self-hosted DLP gateway between your AI agents and LLM providers. Every request is scanned for secrets, PII and stop-words — before it leaves your network.

How it works
Your agents
Claude Code, CI bots, internal apps — anything speaking the OpenAI-style API.
ci-agent dev-ivan
→
Aperture
Every request passes through the focus: scanned, redacted or blocked, logged.
block redact allow
→
Providers
Only clean traffic reaches the cloud.
OpenAI Anthropic Groq

Connect in one line

No SDK, no sidecar agents. Point your existing client at Aperture and keep working — the API surface is identical.

client.py diff 
  client = OpenAI(
-   base_url="https://api.openai.com/v1",
+   base_url="http://aperture.internal:8080/v1",
    api_key=os.environ["APERTURE_KEY"],
  )
DLP inspection
AWS keys, tokens, private keys, cards, emails, custom stop-words. Block, redact or alert — per key, per rule group.
Incident log
Who, when, and what almost left the building — with masked samples. Evidence without exposure.
Cost & latency tracking
Tokens, spend and latency per model and per key. Know which agent burns the budget.

Your data never leaves your network

Aperture runs inside your perimeter as a single Go binary. Logs, incidents and policies stay on your disk. No telemetry, no phone-home, no third-party storage.

$ ./aperture --listen :8080
